## 方案对比

### 方案概览

| 方案 | 核心思路 | 代表工具 |
|---|---|---|
| **A. Blender 全流程** | 手工建模+绑定+动画 | Blender 4.x |
| **B. MetaHuman 改型** | MetaHuman 基础 + 改 Q版 | UE5 MetaHuman Creator |
| **C. AI 生成基础 + 手工修正** | AI 生成基础模型 + Blender 修正 | Meshy / Tripo / Rodin + Blender |
| **D. Mixamo 动画 + 手工模型** | 模型自己做，动画用 Mixamo | Mixamo + Blender |
| **E. 混合管线 (推荐)** | AI 生成 + Blender 修正 + Mixamo 动画 + UE5 Retarget | 多工具链 |

---

### 方案 A: Blender 全流程

```
Blender 建模 → 拓扑优化 → UV → 贴图 → 骨骼绑定 → 动画 K 帧 → 导出 FBX → UE5
```

| 维度 | 评分 | 说明 |
|---|---|---|
| 风格匹配 | ★★★★★ | 完全可控，完美实现 Q版三头身 |
| 质量 | ★★★★★ | 手工品质最高 |
| 速度 | ★☆☆☆☆ | 每职业 2-4 周（建模+动画） |
| 技能门槛 | ★☆☆☆☆ | 需专业 3D 建模+动画技能 |
| 可迭代 | ★★★☆☆ | 修改成本高 |
| 成本 | ★★☆☆☆ | 时间成本高，或外包成本高 |

**每职业工作量估算**:
- 建模: 40-80 小时
- 贴图: 20-40 小时
- 绑定: 8-16 小时
- 动画 (15 个): 80-160 小时
- **总计: 150-300 小时/职业**
- **3 职业 = 450-900 小时**（仅角色，不含敌人/场景）

**适用场景**: 预算充足、有专业美术、追求最高品质

---

### 方案 B: MetaHuman 改型

```
MetaHuman Creator (基础角色) → UE5 Retarget (调整比例) → Blender 调整 → NPR 材质
```

| 维度 | 评分 | 说明 |
|---|---|---|
| 风格匹配 | ★★☆☆☆ | MetaHuman 偏写实，改 Q版 困难 |
| 质量 | ★★★★☆ | 基础质量高，但风格调整受限 |
| 速度 | ★★★☆☆ | 基础角色 1-2 天，调整 1-2 周 |
| 技能门槛 | ★★★☆☆ | 需要 UE5 + 基本 Blender 技能 |
| 可迭代 | ★★★★☆ | MetaHuman 参数可快速调整 |
| 成本 | ★★★★☆ | 快速起步，后续调整成本可控 |

**主要问题**:
- ⚠️ MetaHuman 骨骼基于真人比例，改 Q版 三头身需大幅 retarget
- ⚠️ 面部 BlendShape 偏写实，Q版 表情支持差
- ⚠️ 材质系统偏写实 PBR，NPR/卡通渲染需自定义 Shader
- ⚠️ 体型修改受限，很难做到"大头小身"比例

**适用场景**: 写实/半写实风格，不需要夸张 Q版

---

### 方案 C: AI 生成基础 + 手工修正

```
AI 3D 生成 (Meshy/Tripo/Rodin) → Blender 修正拓扑/UV → 绑定 + 动画 → UE5
```

| 维度 | 评分 | 说明 |
|---|---|---|
| 风格匹配 | ★★★☆☆ | AI 可生成风格化模型，但一致性需控制 |
| 质量 | ★★★☆☆ | 基础质量中等，需手工修正 |
| 速度 | ★★★★☆ | AI 生成 5-30 分钟/模型 |
| 技能门槛 | ★★★☆☆ | AI 操作 + Blender 基础修正 |
| 可迭代 | ★★★★☆ | 可快速重新生成 |
| 成本 | ★★★★★ | 时间+费用都较低 |

**2026 年 AI 3D 工具现状**:

| 工具 | 强项 | 局限 | 费用 |
|---|---|---|---|
| **Meshy.ai** | 风格化模型生成、PBR 贴图 | 拓扑需修正、动画支持有限 | $20/月起 |
| **Tripo3D** | 快速生成基础模型 | 细节不足、需大量后处理 | $10/月起 |
| **Rodin (Hyper)** | 人物模型质量较高 | 风格化支持一般 | $30/月起 |
| **CSM (Common Sense Machines)** | 图像转 3D | 质量不稳定 | 研究阶段 |
| **Tripo / Meshy 动画** | 基础动画自动生成 | 复杂动作质量差 | 新特性 |

**关键风险**:
- ⚠️ AI 模型拓扑通常不适合游戏（需重拓扑）
- ⚠️ 角色一致性难以保证（同一职业不同视角可能差异大）
- ⚠️ 动画 AI 仍在早期，复杂战斗动画质量不够
- ⚠️ 风格化（Q版）支持有限，多数 AI 偏写实

**适用场景**: 快速原型、场景道具、风格化要求不高的资产

### 方案 D: Mixamo 动画 + 手工模型

```
Blender 建模 → Mixamo 自动绑定 → Mixamo 动画库 → UE5 Retarget → 微调
```

| 维度 | 评分 | 说明 |
|---|---|---|
| 风格匹配 | ★★★★☆ | 模型可完全控制，动画质量中上 |
| 质量 | ★★★★☆ | 模型高质+动画流畅 |
| 速度 | ★★★☆☆ | 建模慢，但动画快 |
| 技能门槛 | ★★☆☆☆ | 需建模技能，动画技能要求低 |
| 可迭代 | ★★★☆☆ | 模型修改成本中等 |
| 成本 | ★★★☆☆ | 时间+Mixamo 订阅 |

**Mixamo 关键能力**:
- ✅ 自动骨骼绑定（上传模型即可）
- ✅ 2000+ 预制动画（走/跑/跳/攻击/死亡等）
- ✅ 动画参数可调（速度/幅度）
- ✅ 免费使用（Adobe 账户）
- ⚠️ 风格偏写实（Q版 动画需 retarget 后调整）
- ⚠️ 战斗特定动画较少（需要定制）

**适用场景**: 有建模能力但缺动画能力的团队

### 方案 E: 混合管线 (推荐) ⭐

```
┌─────────────────────────────────────────────────────┐
│                    混合管线流程                       │
├─────────────────────────────────────────────────────┤
│                                                      │
│  阶段 1: 参考图 & 概念                               │
│  ├─ 复用已有 2D 定稿 (wan2.7-image-pro 生成)          │
│  ├─ AI 生成 3D 基础 (Meshy/Tripo, 角色参考)           │
│  └─ 输出: 多视图参考图 + 粗糙 3D 基础模型              │
│                                                      │
│  阶段 2: 角色建模 (Blender)                          │
│  ├─ 基于参考图手工建模 (或修正 AI 基础)                │
│  ├─ 拓扑优化 (游戏级拓扑, < 10k 三角面)               │
│  ├─ UV 展开 + 手绘/程序化贴图                         │
│  └─ 输出: 游戏级 Mesh + Texture                      │
│                                                      │
│  阶段 3: 骨骼绑定                                     │
│  ├─ Mixamo 自动绑定 (快速原型)                        │
│  ├─ 或 Blender 手工绑定 (精细控制)                    │
│  ├─ Q版比例调整 (骨骼缩短, 大头适配)                  │
│  └─ 输出: Rigged Mesh                                │
│                                                      │
│  阶段 4: 动画制作                                     │
│  ├─ Mixamo 基础动画 (walk/run/jump/idle)              │
│  ├─ UE5 IK Retarget (从 Mixamo 到自定义骨骼)          │
│  ├─ Blender 手工动画 (战斗/技能/必杀 — 核心手感)       │
│  ├─ AI 动画生成 (辅助, 待验证质量)                    │
│  └─ 输出: 动画集合                                   │
│                                                      │
│  阶段 5: UE5 集成                                     │
│  ├─ 导入 FBX/glTF → UE5 SkeletalMesh                 │
│  ├─ NPR 卡通渲染材质 (Custom Shader)                  │
│  ├─ Animation Blueprint (状态机 + BlendSpace)         │
│  ├─ 动画 Retarget (UE5 IK Retargeter)                │
│  └─ 输出: 完整可玩角色                               │
│                                                      │
└─────────────────────────────────────────────────────┘
```

| 维度 | 评分 | 说明 |
|---|---|---|
| 风格匹配 | ★★★★★ | Blender 完全控制 Q版 风格 |
| 质量 | ★★★★☆ | 模型手工精修 + 动画 Mixamo/手工结合 |
| 速度 | ★★★★☆ | AI 辅助 + Mixamo 加速, 比纯手工快 3-5× |
| 技能门槛 | ★★★☆☆ | 需要 Blender 基础, 不要求专业动画师 |
| 可迭代 | ★★★★☆ | 各阶段独立, 可单独修改 |
| 成本 | ★★★★★ | 时间+费用都合理 |

**每职业工作量估算 (方案 E)**:
- 阶段 1 (参考): 2-4 小时
- 阶段 2 (建模): 16-32 小时 (AI 辅助可降至 8-16 小时)
- 阶段 3 (绑定): 4-8 小时 (Mixamo 自动绑定 1 小时)
- 阶段 4 (动画): 40-80 小时 (Mixamo 基础 + 手工战斗)
- 阶段 5 (UE5): 16-24 小时
- **总计: 80-150 小时/职业**
- **3 职业 = 240-450 小时**（比方案 A 节省 50%）

### P0 原型验证（最关键）

目标：用最小成本验证整个管线可行性和美术风格

```
第 1 周: 风格探索
├─ 用 Meshy/Tripo 生成 5-10 个 Q版 角色草稿
├─ 选 1 个最接近的作为基础
├─ Blender 调整为准确的 Q版三头身比例
└─ 测试 NPR 卡通渲染效果

第 2 周: 绑定 & 基础动画
├─ Mixamo 自动绑定
├─ 导入 Mixamo 基础动画 (idle/walk/run/jump)
├─ UE5 IK Retarget 测试
└─ 验证骨骼比例是否适配 Q版

第 3 周: 战斗动画
├─ 手工制作 3 个关键战斗动画 (Light/Heavy/Special)
├─ 在 UE5 中接入 GAS 测试
└─ 评估动画质量和手感
```

**P0 成功标准**:
- [ ] Q版三头身比例准确
- [ ] NPR 卡通渲染效果满意
- [ ] 基础移动流畅
- [ ] 战斗动画有打击感
- [ ] 总工期 < 3 周

### NPR 卡通渲染方案

UE5 的 NPR 选项：

| 方案 | 效果 | 复杂度 | 推荐 |
|---|---|---|---|
| **UE5 Toon Shader** | 标准卡通渲染 | 低 | ✅ 首选 |
| **UE5 Stylized Rendering** | 风格化渲染 | 中 | 适合特殊效果 |
| **Custom NPR Material** | 完全自定义 | 高 | 风格独特时 |
| **PostProcess Cel-Shade** | 后期处理描边 | 低 | 配合使用 |

崩坏3 类似风格的 NPR 实现参考：
- 主色调 + 阴影色 (2-tone shading)
- 描边 (Outline) via Inverted Hull 或 PostProcess
- 高光 (Rim Light) 强化轮廓
- 自发光 (Emission) 用于技能特效

## 决策建议

### 立即可做的

1. **启动 P0 原型** — 用最小成本验证整个管线
   - 安装 Blender + Mixamo 账户
   - 用 AI 工具（Meshy 试用）生成 Q版 草稿
   - 3 周内产出第一个可动角色

2. **同时探索 NPR 渲染** — 在 UE5 中实验卡通渲染
   - 参考崩坏3/原神 的材质设置
   - 用简单立方体测试 NPR 效果
   - 确定最终渲染方案

3. **建立资产规范** — 面数、贴图尺寸、骨骼命名等

### 需要决定的

| 决策 | 建议 | 决策时间 |
|---|---|---|
| AI 工具选择 | Meshy 试用 → 决定 | P0 阶段 |
| Mixamo vs 手工绑定 | Mixamo 优先（速度） | P0 阶段 |
| NPR 方案 | Toon Shader 首选 | P0 阶段 |
| 外包 vs 自制 | MVP 自制，后续看情况 | P1 阶段 |

### 不建议做的

- ❌ **不建议用 MetaHuman** — Q版 三头身风格不匹配
- ❌ **不建议纯 AI 生成** — 质量不够稳定
- ❌ **不建议纯手工全流程** — 工期太长（solo 不现实）
- ❌ **不建议购买现成资产** — 风格难统一
