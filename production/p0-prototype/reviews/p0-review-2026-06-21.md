##  技术风险与发现

### UE5.7 Python API 限制

| 限制 | 影响 |  workaround |
|------|------|-------------|
| `MaterialInstanceConstantFactoryNew` 无 `Parent` 属性 | 无法通过 Python 创建材质实例 | 使用 C++ 或 Editor GUI |
| 运行时材质保存失败 | 无法动态创建并保存材质 | 预创建材质资产 |
| `-RunPythonScript` + `-nullrhi` 崩溃 | 无法在无渲染模式下运行 Python | 使用 `-game` 模式或完整编辑器 |

### MCP 材质编辑限制

| 限制 | 影响 | 解决方案 |
|------|------|----------|
| Python 可读取材质属性但无法修改 | 无法通过 MCP 构建材质图 | 使用 Editor GUI |
| MaterialEditingLibrary 调用失败 | 无法程序化编辑材质 | 手动编辑 |
| manage_asset 材质编辑无响应 | 工具不支持复杂材质操作 | 等待插件更新 |

### C++ 材质图序列化问题

| 问题 | 根因 | 状态 |
|------|------|------|
| `UMaterialExpressionIf.AGreaterThanB` 连接丢失 | UE5.7 序列化变更 | 已用 Step 节点替代 |
| `UMaterialExpressionOneMinus.Input` 连接丢失 | 同上 | 已用 Add+Constant 替代 |
| 材质保存时文件移动失败 | 包路径问题 | 改用瞬态材质 (不保存) |
